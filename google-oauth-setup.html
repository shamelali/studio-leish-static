Studio Leish

Google OAuth Setup Guide

Step 1: Google Cloud Console

1.1 Create Project

Go to Google Cloud Console

Create a new project or select existing one.

1.2 Enable APIs

Navigate to APIs & Services → Library

Search for Google+ API and enable it.

Step 2: OAuth Consent Screen

2.1 Configure Consent Screen

Go to OAuth Consent Screen in Google Cloud Console

Select External user type

Fill in required fields:

App name: Studio Leish
User support email: admin@leish.my
Developer contact: admin@leish.my

2.2 Add Scopes

Add scopes:

.../auth/userinfo.email
.../auth/userinfo.profile
openid

Step 3: Create OAuth Credentials

3.1 Create Client ID

Go to Credentials → Create Credentials → OAuth Client ID

Select application type: Web Application

3.2 Configure Redirect URIs

Authorized Redirect URIs:

  • https://studio-leish-static.vercel.app/auth/callback
  • https://studio.leish.my/auth/callback (after domain setup)
  • http://localhost:3000/auth/callback (for local dev)

3.3 Copy Credentials

After creation, copy:

Client ID: YOUR_GOOGLE_CLIENT_ID.apps.googleusercontent.com
Client Secret: YOUR_GOOGLE_CLIENT_SECRET

Step 4: Configure Supabase Auth

4.1 Add Provider in Supabase

Go to Supabase Dashboard

Navigate to Authentication → Providers → Google

Click Enable Google provider

4.2 Enter Credentials

Enter the credentials from Step 3.3:

Client ID: [Your Google Client ID]
Client Secret: [Your Google Client Secret]
Redirect URL: https://vnpiekiedynvechawhhw.supabase.co/auth/v1/callback
Important: The Redirect URL in Supabase should auto-populate. Copy it and add to Google Cloud Console in Step 3.2!

Step 5: Update Application

5.1 Add Environment Variables

Add to Vercel environment variables:

VERCEL env add NEXT_PUBLIC_GOOGLE_CLIENT_ID
VERCEL env add GOOGLE_CLIENT_SECRET

Values:

NEXT_PUBLIC_GOOGLE_CLIENT_ID=YOUR_GOOGLE_CLIENT_ID
GOOGLE_CLIENT_SECRET=YOUR_GOOGLE_CLIENT_SECRET

5.2 Update user.html

Add Google Sign-In button to user.html:

<button onclick="signInWithGoogle()">
  <i class="fab fa-google"></i> Sign in with Google
</button>

Add JavaScript function:

async function signInWithGoogle() {
  const { data, error } = await supabase.auth.signInWithOAuth();
  if (error) console.error('Google OAuth error:', error);
}

Step 6: Test Google OAuth

6.1 Test Flow

Visit user.html and click "Sign in with Google"

Should redirect to Google login → back to app

✅ Success! Google OAuth is now integrated with Studio Leish!
⚠️ Important Notes:
1. Google OAuth requires HTTPS in production (already handled by Vercel + Cloudflare)
2. Make sure to add ALL redirect URIs to Google Cloud Console
3. Test with real Google account (not just test accounts)
4. Verify email verification is enabled in Supabase Auth settings
